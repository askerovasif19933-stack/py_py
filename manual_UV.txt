Удаление
Чтобы удалить UV, нужно выполнить следующие команды

uv cache clean
rm -r "$(uv python dir)"
rm -r "$(uv tool dir)"

                  
Удалить оставшиеся файлы кэширования

rm ~/.local/bin/uv ~/.local/bin/uvx


                  
Для Windows

$ rm $HOME\.local\bin\uv.exe
$ rm $HOME\.local\bin\uvx.exe

Команда установки Python выглядит следующим образом:

$ uv python install

Если нужно установить конкретную версию Python, то нужно сначала проверить список доступных версий в репозитории, а затем установить нужную:

$ uv python list 
$ uv python install 3.12 


Команда удаления интерпретатора python выглядит следующим образом:

$ uv python uninstall

Чтобы обновить установленные версии, существует команда:

 $ uv python install --reinstall 

 Команда удаления интерпретатора python выглядит следующим образом:

$ uv python uninstall

И, наконец, проверить, где находится текущий интерпретатор python можно командой:

$ uv python find

Чтобы создать новое виртуальное окружение в директории my_dir и переключиться на него, нужно ввести команды:

$ cd my_dir
$ uv venv - создаёт новое виртуальное окружение в директории .venv 
$ source ./.venv/bin/activate
Также можно указать название директории, в которой будет создано окружение, например, .my_env:

$ uv venv .my_env

Теперь можем проверить успешную активацию окружения уже изученной командой:

$ uv python find

Выведет путь до интерпретатора python внутри директории my_dir/.venv:

 /home/alexander/development/my_dir/.venv/bin/python3

 

Команда активации окружения для Windows Powershell:

> .venv\Scripts\activate

 

Когда вы выполняете команды вроде uv add requests или uv run python app.py, UV сам создаёт (если нужно) виртуальное окружение и использует его. То есть, вам не нужно заботиться о python -m venv и не нужно активировать окружение вручную - UV делает это за вас.

 

Следует отметить, что активировать виртуальное окружение обычно нет необходимости - все команды uv будут выполняться с учётом созданного окружения автоматически. Чтобы проверить это, можно запустить команды:

$ deactivate
$ uv python find
В результате будет выведен тот же путь до интерпретатора python в директории my_dir


uv run python - запуск python
uv run app.py - запуск скрипта
uv run uvicorn main:app --port 8080 --reload  - запуск приложения fastapi через uvicorn
Если пакета uvicorn ещё нет — UV установит его автоматически перед запуском.

выберем сложный путь, и выполним следующие команды:

uv python pin 3.12
uv venv
uv init --bare
uv add fastapi[standard] uvicorn[standard]

                  
Здесь:

uv init --bare создаёт файл pyproject.toml, необходимый для установки зависимостей

 

Файл pyproject.toml хранит декларацию зависимостей (например, "fastapi>=0.115"), а uv.lock - заморозку конкретных версий (например, "fastapi==0.115.0"). Таким образом, pyproject.toml - это то, что вы хотите,
а uv.lock - то, что вы получили
